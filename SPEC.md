# Flux Bot - Спецификация

## 1. Обзор проекта

**Название**: Flux Bot
**Тип**: Telegram бот для гарантирования сделок
**Основная функция**: Выступает гарантом при сделках между пользователями, обеспечивая безопасную передачу товаров и оплаты.

## 2. Типы сделок

### 2.1 Обычная сделка (Товар)
- Продавец создает сделку с указанием товара и цены
- Покупатель оплачивает (звезды или валюта) боту
- Продавец передает товар покупателю
- Покупатель подтверждает получение
- Админ переводит деньги продавцу (или уведомляется при валюте)

### 2.2 Сделка Telegram Gift (Подарок)
- Покупатель оплачивает (звезды или валюта) боту
- Продавец передает подарок покупателю напрямую в Telegram
- Покупатель подтверждает "Это тот подарок" или открывает спор
- Админ переводит деньги продавцу (или уведомляется при валюте)

## 3. Технологический стек

- **Язык**: Python 3.12+
- **Фреймворк**: aiogram 3.x
- **База данных**: aiosqlite
- **Оплата**: Telegram Stars / Валюта (RUB, USD, EUR)

## 5. Функциональность

### 5.1 Главное меню

Кнопки:
1. **Добавить реквизиты карты** - добавить/изменить реквизиты
2. **Создать сделку** - начать создание новой сделки
3. **Мои сделки** - просмотр своих сделок
4. **Реферальная ссылка** - получить реферальную ссылку
5. **Поддержка** - связаться с поддержкой

### 5.2 Создание сделки

Продавец выбирает:
1. **Тип сделки**: Обычный товар или Telegram Gift
2. **Название товара**: текстовое описание
3. **Способ оплаты**: Звезды или Валюта
4. **Валюта** (если выбрана): RUB, USD, EUR
5. **Цена**: сумма в звездах или валюте

После создания бот генерирует ссылку: `t.me/BotUsername?start=deal_{deal_id}`

Требование: продавец должен иметь добавленные реквизиты карты для создания сделки.

### 5.3 Присоединение к сделке

Покупатель видит:
- Информацию о сделке (товар, цена, продавец)
- Кнопку "Присоединиться к сделке"

После присоединения продавец получает уведомление с данными покупателя.

### 5.4 Оплата

Покупатель нажимает "Я оплатил" - деньги поступают боту (имитация, для реальной оплаты требуется Telegram Payments API).

При оплате валютой админ получает уведомление для ручного перевода.

### 5.5 Завершение сделки (Товар)

1. Покупатель оплатил
2. Продавец нажимает "Я передал товар"
3. Покупатель подтверждает "Я получил товар" или открывает спор
4. Сделка завершена, админ переводит деньги

### 5.6 Завершение сделки (Gift)

1. Покупатель оплатил
2. Продавец нажимает "Я передал подарок"
3. Продавец отправляет подарок покупателю напрямую в Telegram
4. Покупатель подтверждает "Это тот подарок" или "Это не тот подарок"
5. При подтверждении - сделка завершена
6. При отказе - покупатель описывает проблему с доказательствами
7. Продавец может ответить с доказательствами
8. Админ разрешает спор

### 5.7 Система споров

При открытии спора:
- Покупатель отправляет текст/фото/документ с описанием проблемы
- Продавец получает уведомление и может ответить с доказательствами
- Админ видит доказательства обеих сторон
- Админ решает: вернуть деньги покупателю или выплатить продавцу

При нажатии "Отменить" после спора покупатель возвращается к подтверждению.

### 5.8 Реферальная система

- При старте с `start=USER_ID` - пользователь становится рефералом
- При успешной сделке реферер получает 1% от цены в звездах

### 5.9 Админ-панель

Доступ через `/admin`:
1. **Все сделки** - просмотр всех сделок
2. **Сделки пользователя** - просмотр сделок конкретного пользователя
3. **Бан пользователя** - бан по telegram_id

При завершении сделки админ видит реквизиты продавца.
При отмене сделки админ видит реквизиты покупателя.

### 5.10 Автоудаление

Сделки со статусом pending/joined старше 24 часов автоматически удаляются.

## 6. Состояния сделки (State Machine)

```
pending -> joined -> paid -> gift_sent -> gift_received -> completed
                \-> cancelled
                \-> disputed
```

- **pending**: Ожидает присоединения покупателя
- **joined**: Покупатель присоединился
- **paid**: Оплата получена (для товара)
- **gift_sent**: Продавец подтвердил передачу подарка
- **gift_received**: Подарок отправлен покупателю
- **completed**: Сделка завершена
- **cancelled**: Сделка отменена
- **disputed**: Спор - требуется вмешательство админа

## 7. Валидация

- Номер счета/телефона: от 5 до 25 цифр
- Проверка наличия реквизитов карты при создании сделки
